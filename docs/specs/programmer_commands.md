# Программатор v0 — команды и форматы

Этот документ описывает текущие возможности “Программатора” в `meditation_poc.py`.

## 1) Spec (одна строка)

`Spec` — это строка из токенов, разделённых пробелами.

Поддерживаемые токены:

### 1.1 Binaural (стерео)

Формат:

`<carrier_hz>+<beat_hz>/<amp>`

Где:
- **carrier_hz**: несущая частота (Гц)
- **beat_hz**: биений (Гц), будет разнесено по ушам как \(carrier \pm beat/2\)
- **amp**: амплитуда/громкость (см. “Амплитуда”)

Примеры:
- `100+7/0.60`
- `250+4/35` (35% → 0.35)

### 1.2 Noise (фон)

Формат:

`white/<amp>` или `pink/<amp>` или `brown/<amp>`

Примеры:
- `pink/0.08`
- `brown/12` (12% → 0.12)

### 1.3 Sweep (тон-свип)

Формат:

`sweep:<f0_hz>-><f1_hz>/<dur_s>/<amp>`

Примеры:
- `sweep:1000->100/30/0.6`
- `sweep:400->220/10/40` (40% → 0.40)

### 1.4 Выключение

Формат:
- `off`
- `-`

### 1.5 Комбинирование

Можно объединять токены:

`100+7/0.60 pink/0.08`

`sweep:1000->100/30/0.6 brown/0.10`

## 2) Timeline (блокнот)

`Timeline` — текст из строк формата:

`<mm:ss> <spec>`

или:

`<hh:mm:ss> <spec>`

Правила:
- пустые строки игнорируются
- комментарии начинаются с `#` (можно в конце строки)
- строки сортируются по времени и планируются на исполнение

Пример:

```text
0:00 sweep:1000->100/30/0.6 pink/0.08
0:25 100+7/0.60 pink/0.08
0:30 off
```

## 3) Амплитуда (amp)

Везде, где используется `<amp>`:
- **0..1** — воспринимается как “нормированная” амплитуда
- **> 1** — воспринимается как “проценты” и делится на 100

То есть:
- `0.60` == 60%
- `60` == 60%

## 4) Agent API (внешнее управление)

Если в UI включен `Agent API :8765`, то можно управлять программатором HTTP запросами.

Endpoint:
- `POST http://127.0.0.1:8765/v1/event`

Тело:

```json
{"topic":"...", "payload":{...}}
```

### 4.1 Установить Spec

```json
{
  "topic": "program.set_spec",
  "payload": { "spec": "100+7/0.60 pink/0.08" }
}
```

### 4.2 Установить Timeline

```json
{
  "topic": "program.set_timeline",
  "payload": {
    "timeline": "0:00 sweep:1000->100/30/0.6 pink/0.08\n0:25 100+7/0.60 pink/0.08\n0:30 off\n"
  }
}
```

### 4.3 Stop

```json
{ "topic": "program.stop", "payload": {} }
```

## 5) Status и Sink URL

UI публикует в шину событие:
- `program.status`

Если заполнено поле `Sink URL`, UI будет POST-ить на него события формата:

```json
{ "topic": "program.status", "payload": { ... } }
```

Частота отправки ограничена примерно до 4 Гц (throttle).
